## markup_system_rules

General semantic HTML markup rules for mark-based annotations: allowed tags, required `class="highlight"` and `data-type` attributes, constraints on overlapping types, and the separation of semantic markup from visual styling (delegating all CSS to the frontend). Encodes the one-type-per-mark rule and disallows inline styles or Markdown highlighting syntax.
