## highlight_formatting_rules

Defines how gaps, unverified information, and uncertain content are marked using `<mark class="highlight">` with appropriate `data-type` values (`gap`, `citation`, or default). It bans inline styles and Markdown `==...==` syntax and provides canonical examples for `GAP`, `TBD`, and generic “need to confirm” highlights.
