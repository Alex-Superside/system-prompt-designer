## brief_allowed_html_tags_rules

Restricts the set of HTML tags the brief-writing agent may emit (`<h1>`–`<h3>`, `<p>`, `<ul>`, `<li>`, `<mark>`, `<sup>`, `<br>`, `<hr>`) and clarifies that no other tags, inline styles, or Markdown should be present. This allows the frontend to rely on a constrained, predictable HTML subset.
