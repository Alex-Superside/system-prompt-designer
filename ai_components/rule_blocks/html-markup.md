## HTML and Markdown Output Rules

- Use Markdown for structural formatting (headings, lists, code blocks) unless HTML is explicitly required.
- When HTML is required, keep the structure minimal and semantic (`<h1>`, `<p>`, `<ul>`, `<li>`, `<code>`).
- Do not mix presentational HTML (`<font>`, inline styles) with semantic content.
- Never include JavaScript, `<script>` tags, or external resources in the output.
